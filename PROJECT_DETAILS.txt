PROJECT DETAILS
================

Project name:
Drought Risk Mapping for the MENA Region

Purpose:
This project demonstrates a geospatial machine-learning workflow for drought analysis across the Middle East and North Africa. It is positioned as a portfolio project that combines satellite data, climate variables, feature engineering, classification, and map-based communication.

Main conclusion from the project check:
The project already has strong content, but it needed better structure, clearer entry points, and supporting files that make it easier to run, explain, and share professionally.

---------------------------------------------

The folder originally contained:

- 4 notebooks with overlapping purpose
- 1 README draft with a strong description but a repo structure that did not yet exist

- 1 markdown note about setting up a GitHub repository

2. MAIN ISSUES FOUND
--------------------

Issue 1:
The repository did not have standard filenames like README.md, requirements.txt, and .gitignore, even though drafts for them existed.

Issue 2:
The documentation described a polished project structure with folders like outputs/, data/, and assets/screenshots/, but those folders were not actually present.

Issue 3:
There were multiple notebooks with similar names and overlapping content, which makes it harder for another person to know which notebook to run first.

Issue 4:
The real-data notebook was the strongest version, but it did not yet fully standardize export steps for figures, maps, and tables inside the notebook itself.

Issue 5:
There was no quick utility script to bootstrap directories or verify that the Python environment is ready.

Issue 6:
This is not currently a git repository, so version control still needs to be initialized if you want to publish or track changes cleanly.

5. WHAT THE MAIN NOTEBOOK DOES
------------------------------

High-level workflow:

1. Initialize the environment and Earth Engine.
2. Define the MENA study area using a broad bounding box.
3. Build a rectangular fishnet grid.
4. Load MODIS NDVI, CHIRPS rainfall, and MODIS land surface temperature.
5. Aggregate monthly variables over the grid.
6. Convert the Earth Engine results into a pandas DataFrame.
7. Engineer climatology and anomaly features.
8. Create drought classes using NDVI anomaly thresholds.
9. Train a Random Forest classifier.
10. Visualize model results and map the latest predictions with Folium.

Main datasets referenced:

- MODIS/061/MOD13A3 for NDVI
- UCSB-CHG/CHIRPS/DAILY for rainfall
- MODIS/061/MOD11A2 for land surface temperature

Main machine-learning approach:

- RandomForestClassifier from scikit-learn

Main mapping approach:

- folium.CircleMarker on the latest model month

7. WHAT IS STILL WEAK OR INCOMPLETE
-----------------------------------

- The broad bounding-box grid is useful for a prototype, but it is not yet policy-grade geography.
- The notebook now exports key portfolio assets, but the broader workflow is still notebook-centric rather than a reusable package or pipeline.
- The drought labels are still anomaly-rule proxies rather than independent observed event labels.
- Validation is basic and could be improved with stronger temporal or spatial testing.
- Multiple notebooks still create ambiguity about the final intended version.


8. HOW TO RUN THE PROJECT
-------------------------

Recommended steps:

1. Install Python 3.10 or newer.
2. Create a virtual environment.
3. Install dependencies from requirements.txt.
4. Run scripts/project_setup.py.
5. Run scripts/environment_check.py.
6. Launch Jupyter Notebook or JupyterLab.
7. Open drought_risk_mapping_MENA_real_data_portfolio.ipynb.
8. Authenticate Google Earth Engine if needed.
9. Run the notebook from top to bottom.

Expected output locations:

- outputs/figures/
- outputs/maps/
- outputs/tables/
- assets/screenshots/
